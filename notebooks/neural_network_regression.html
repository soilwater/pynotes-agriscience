Neural Network Regression¶

Neural networks are a type fo supervised learning. This means that to get started we need to have a set of explanatory variables and the expected result or outcome (the target varaible). This is called a training set, and they are usually obtained from databases, experiments, and dedicated data collection campaigns. In simple terms, we can think of a neural network as a sophisticated interpolation approach. So, if we want the neural network to produce results within plausible limits of the target varaible we need to provide a wide range of input scenarios, so that the network “learns” what the target is not only in common scenarios for which we will likely have lots of data, but also in less common situation. This means lots of data.

Fortunately, environemtnal moniotring networks collect abundant atmospheric and soil observations every day. In this notebook we will use weather variables to predict reference evapotranspiration.

The Penman-Monteith grass reference evapotrasnpiration (ETo) requires several variables and numerous equations. In this exercise we will use the same variables that we would use to compute the ETo by the PM method. Our dataset already has pre-computed ETo values for grass. The goal is to test whether feeding the same variables to a neural network would result in comparable performance to the PM method.

Of course a neural network would be of little use if we have the PM model already implemented. However, it is a good exercise to examine the capacbilities of simple multi-perceptron networks. This naive example, however, could be useful for a researcher to predict ETo based on an instrument that can measure ETo (i.e. a lysimeter, basically a heavy duty scale exposed tot he elements that records the mass of the soil undergoing evapotranspiration) co-located with weather variables. In such case, the scientist could develop a neural network to predict the measurements by the instrument given the weather data, without having to rely on the PM method.

Hopefully this simple exercise will be sufficient to get you started with your own data.

Load dataset¶

We will use weather data from the Acme station of the Oklahoma Mesonet. The dataset contains the following variables from 2005 to 2017:

  • Date: Daily timestamp

  • DOY: Day of the year

  • TMAX: Maximum daily air temperature (Celisus)

  • TMIN: Minimum daily air temperature (Celisus)

  • HMAX: Maximum daily air relative humidity (%)

  • HMIN: Minimum daily air relative humidity (%)

  • RAIN: Daily rainfall total (mm). This variable is not use in this example

  • ATOT: Total daily incident solar radiation (\(MJ/m^2\))

  • W2AVG: Daily average wind speed at 2 meters height (\(m/s\))

  • ETgrass: Daily reference evapotranspiration for a short grass (\(mm/day\))

Define inputs and outpus¶

The enxt step is to let the model know which are the explanatory variables (X) and which is/are the target variables (y). The Pandas library makes this step super easy. We only need to select the columns by name.

# Select predicting (explanatory) variables
X = df[['TMAX','TMIN','HMAX','HMIN','ATOT','W2AVG']]

# Select target variables
y = df['ETgrass']

Train and test sets¶

We need to train our model in a subset of the original dataset that we loaded and then test it using an independent set of samples (the test set). The goal is to test the model in subsamples that the model has never seen. In other words, after training, we will test the performance of the test model using a new subset of unseen samples.

The scikit-learn library provide some functions that allow us to easily partition the X and y entries into a training and testing subsets. By default the function train_test_split will split the dataset into a training subset with 75% of the observations and a test set with the remaining 25% of the observations. The function shuffles the entries in X, but keeps the columns for each row together. the function also splits and shuffles the matching rows of y, so that inputs and targets remain aligned. In other words, each row represents an individual entry. You can clearly see this below when we print the different subsets (which happen to be Pandas Dataframes), look at the index number

# Create train and test sets
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=1)

Normalize data¶

This step is critical to ensure that each input variable has the same weight at the time of training. The normalization of the training set and the test set need be performed after splitting the main dataset, so that the test set remains fully independent of the training set. In some cases, good performance can be obtained without normalization, but normalization rarely hurts the training process.

X_train_normalized = StandardScaler().fit_transform(X_train)
X_test_normalized = StandardScaler().fit_transform(X_test)

Fit the model¶

We now simply need to pass the inputs and targets into the fitting function. This is the part that while it is not necessary to know in order to fit a neural network to your data, it does help tremendously some basic understanding.

Neural networks relies on feedforward pass: Given the provided inputs in a given entry and given some parameters in the neurons the network tries to predict the target output. At the beginning the netowrks starts with some random parameters, so initial predictions are typically bad. However, using these bad guesses and the known targets (recall this is supervised learning), the network computes the diffrences and back propagates this difference into the responsible neurons for most of the error. This is the back-propgation algorith. This step is crucial since it trigger changes in the parameters of the neurons mainly responsible for the error. Once the paramters are updated, a new guess takes place, and subsequently the new error is computed and back-propagated into the neurons. Basically this a trial and error while the network learns how to get the right output. The knowledge of the network consists of the set of parameters in the neurons. A linear model would have two parameters, the intercept and the slope. A neural network can have hundreds or millions of parameters, reason why they can capture such complex patterns. To wrap this basic introduction, in this case we are addressing a regression problem. Our target variable is a continuous variable, but neural networks can also be used to solve classification problems, such as defining whether a set of weather variables would result in rainfall or not (just a yes/no prediction) or whether a set of image pixels represent flowers of a given plant specie.

# Set parameters and then fit model using the training set
regr = MLPRegressor(random_state=1,
                    max_iter=500,
                    hidden_layer_sizes=50, 
                    activation='relu',
                    solver='adam').fit(X_train_normalized, y_train)

# Coefficient of determination of our prediction
regr.score(X_train_normalized, y_train)
